SpryJS

SpryJS

SpryJS is a lightweight and straightforward JavaScript toolkit designed to complement SpryCSS, though it can also be used independently. Its primary objective is to offer essential functionality and utilities that accelerate development.

Support

You can report issues or make feature requests on the GitHub Issues page.

If you like to help us grow, then please add a star on the SpryJS GitHub Repository or by sharing us on social media.

Star Issue Watch

Installation

Using a Package Manager

npm install @ggedde/spry-js
yarn add @ggedde/spry-js
pnpm add @ggedde/spry-js
bun add @ggedde/spry-js

Using the CDN ES Module Build

Using Default Export

<script type="module"> import SpryJS from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; SpryJS('.something').addClass('open'); </script>

Using Partials

<script type="module"> import { cookie, hash, navigable, observe, parallax, scrollspy, slider, toggle, query } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; query('.something').addClass('open'); </script>

Using the CDN Global Build

<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script>

Usage

<script> // Run after dom ready SpryJS('.something').addClass('open'); </script> <!-- OR Inline --> <button onclick="SpryJS('.something').toggleClass('open')"> Click Me </button>

Global Build Query Param Options

load

Automatically load SpryJS to a variable you specify and initialize all components with default options.

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?load=spryJs"></script>

You then have access to your variable when the Dom has been loaded.

<button onclick="spryJs.query('.something').toggleClass('open')"> Click Me </button>

run

Run a function you specify when SpryJS is fully loaded.

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?run=spryJsLoaded"></script> <script> function spryJsLoaded() { SpryJS('.something').addClass('open'); } </script>

Navigable

Navigable enables keyboard navigation for specific elements using the arrow keys and triggerable via the spacebar and enter keys. Additionally, it provides the functionality to use the escape key to dismiss focus.

Usage

<nav class="navigable"> <ul> <li><a href="#">Link 1</a></li> <li><a href="#">Link 2</a></li> <li><a href="#">Link 3</a></li> </ul> </nav>

API

Initiate

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?load=spryJs"></script>
* var spryJsis automatically created and loads all components with default options. You can customize the variable in the src of the script.
<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script> <script> var spryObj = SpryJS('.navigable').navigable(); </script>
<script type="module"> import { navigable } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; const obj = navigable(); </script>
import { navigable } from 'spry-js'; const obj = navigable();

Update

This allows you to update the objects after Dom changes have been made without duplicating the event listeners. Useful with page builders and other dynamic Dom loading applications.

// Update Objects spryJs.navigableUpdate(); // OR Update All Components spryJs.update();
// Update Objects spryObj.navigableUpdate(); // OR Update All Components spryObj.update();
obj.update();
obj.update();

Destroy

This allows you to remove all event listeners and added functionality for the objects.

// Destroy Objects spryJs.navigableDestroy(); // OR Destroy All Components spryJs.destroy();
// Destroy Objects spryObj.navigableDestroy(); // OR Destroy All Components spryObj.destroy();
obj.destroy();
obj.destroy();

Options

navigable({ items: '.navigable', selectorAnchor: 'a, button, input, [tabindex]', attributeAnchor: 'data-navigable-anchors', });
items?: string | Element[]

Selector or array of Elements for navigable containers. This will capture keyboard events within this element and navigate between the anchors.

default: ".navigable"
selectorAnchor?: string

Selector Name for anchor items within the Navigable container. Which items within the Navigable container you wont to be able to navigate through.

default: "a, button, input, [tabindex]"

Override with attributeAnchor
attributeAnchor?: string

Data attribute for custom anchor selector.

default: "data-navigable-anchors"

Overrides selectorAnchor

Observe

Observe enables the tracking of elements based on their position within the viewport. When an element becomes visible, two classes are applied. Upon exiting the viewport, the observing class is removed, while the observed class remains active.

Usage

<div class="observe"></div>

API

Initiate

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?load=spryJs"></script>
* var spryJsis automatically created and loads all components with default options. You can customize the variable in the src of the script.
<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script> <script> var spryObj = SpryJS('.observe').observe(); </script>
<script type="module"> import { observe } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; const obj = observe(); </script>
import { observe } from 'spry-js'; const obj = observe();

Update

This allows you to update the objects after Dom changes have been made without duplicating the event listeners. Useful with page builders and other dynamic Dom loading applications.

// Update Objects spryJs.observeUpdate(); // OR Update All Components spryJs.update();
// Update Objects spryObj.observeUpdate(); // OR Update All Components spryObj.update();
obj.update();
obj.update();

Destroy

This allows you to remove all event listeners and added functionality for the objects.

// Destroy Objects spryJs.observeDestroy(); // OR Destroy All Components spryJs.destroy();
// Destroy Objects spryObj.observeDestroy(); // OR Destroy All Components spryObj.destroy();
obj.destroy();
obj.destroy();

Options

observe({ items: '.observe', rootMargin: '0px 0px 0px 0px', threshold: 0, delay: 50, classObserved: 'observed', classObserving: 'observing', attributeClassObserved: 'data-observe-class-observed', attributeClassObserving: 'data-observe-class-observing', attributeDelay: 'data-observe-delay' });
items?: string | Element[]

Selector or array of Elements to watch in the observer.

default: ".observe"
rootMargin?: string

Bounding box before computing intersections. Negative values will shrink the bounding box.

default: "0px 0px 0px 0px"

Learn more about rootMargin
threshold?: number

Either a single number from 0 - 1 or an array of numbers which indicate at what percentage of the target's visibility the observer's callback should be executed.

default: 0

Learn more about threshold
delay?: number

Delay in milliseconds before the Observer acknowledges the observed element. If multiple elements are observed at the same time then the delay will be stacked by the element index.

default: 100

Override with attributeDelay
classObserved?: string

Class that gets added when elements become visible within the viewport, but does not get removed.

default: "observed"

Override with attributeClassObserved
classObserving?: string

Class that gets toggled when elements are visible within the viewport.

default: "observing"

Override with attributeClassObserving
attributeDelay?: string

Data attribute to specify the delay. This will override delay.

default: "data-observe-delay"

Overrides delay
attributeClassObserved?: string

Data attribute to specify the observed class. This will override classObserved.

default: "data-observe-class-observed"

Overrides classObserved
attributeClassObserving?: string

Data attribute to specify the observing class. This will override classObserving.

default: "data-observe-class-observing"

Overrides classObserving

Parallax

Parallax enables the movement of elements in relation to viewport scroll. By default, this effect is applied to the child element, though it can also be applied to the background position. To function correctly, Parallax requires the child element or background image to overflow the parent container. It is the user's responsibility to manage the visibility of any overflowing content.

Usage

<!-- Parallax Container --> <div style="max-height: 200px; overflow: hidden;"> <!-- Parallax Element --> <div><!-- Large Content Here --></div> </div>

API

Initiate

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?load=spryJs"></script>
* var spryJsis automatically created and loads all components with default options. You can customize the variable in the src of the script.
<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script> <script> var spryObj = SpryJS('.parallax').parallax(); </script>
<script type="module"> import { parallax } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; const obj = parallax(); </script>
import { parallax } from 'spry-js'; const obj = parallax();

Update

This allows you to update the objects after Dom changes have been made without duplicating the event listeners. Useful with page builders and other dynamic Dom loading applications.

// Update Objects spryJs.parallaxUpdate(); // OR Update All Components spryJs.update();
// Update Objects spryObj.parallaxUpdate(); // OR Update All Components spryObj.update();
obj.update();
obj.update();

Destroy

This allows you to remove all event listeners and added functionality for the objects.

// Destroy Objects spryJs.parallaxDestroy(); // OR Destroy All Components spryJs.destroy();
// Destroy Objects spryObj.parallaxDestroy(); // OR Destroy All Components spryObj.destroy();
obj.destroy();
obj.destroy();

Options

parallax({ items: '.parallax', threshold: -300, minWidth: 0, delay: 300, classActive: 'parallaxing', attributeClassActive: 'data-parallax-class-active', attributeBackground: 'data-parallax-background', attributeHorizontal: 'data-parallax-horizontal', attributeInvert: 'data-parallax-invert', attributeDelay: 'data-parallax-delay', });
items?: string | Element[]

Selector or array of Elements to parallax.

default: ".parallax"
threshold?: number

The threshold is the buffer in pixels before the parallax element starts to animate before it reaches the viewport. This helps reduce flickering when the element animates from baseline position. You can use a positive number to prevent animation while partially in viewport.

default: -300
minWidth?: number

Minimum width in pixels of the window to apply effects. Useful if you do not what to apply parallax for smaller devices.

default: 0
delay?: number

Number in milliseconds to apply easing delay. This allows for a smoother animation on non-smooth scrolling devices and might need adjustment based on your needs. Set to 0 to disable.

default: 0

Override with attributeDelay
classActive?: string

Class added to element when being viewed by observer and active. Removed when out of view.

default: ".parallaxing"

Override with attributeClassActive
attributeClassActive?: string

Data attribute to set the active class when the element is being viewed by observer and active.

default: "data-parallax-class-active"

Overrides classActive
attributeBackground?: string

Data attribute to indicate that parallaxing should be done on the "background-position" style instead of the element.

default: "data-parallax-background"
attributeHorizontal?: string

Data attribute to indicate that parallaxing should be done horizontally instead of vertically.

default: "data-parallax-horizontal"
attributeInvert?: string

Data attribute to indicate that parallaxing should be done in the opposite direction.

default: "data-parallax-invert"
attributeDelay?: string

Data attribute that sets the value of the delay time. This will override the delay option if present and allows for different values per element.

default: "data-parallax-delay"

Overrides delay

Scroll Spy

Navigable enables keyboard navigation for specific elements using the arrow keys and triggerable via the spacebar and enter keys. Additionally, it provides the functionality to use the escape key to dismiss focus.

Usage

<!-- ScrollSpy Container --> <div class="scrollspy"> <!-- ScrollSpy Anchors --> <a href="#scrollspy-section-1">Section 1 Anchor</a> <a href="#scrollspy-section-2">Section 2 Anchor</a> <a href="#scrollspy-section-3">Section 3 Anchor</a> </div> <!-- ScrollSpy Sections --> <div id="scrollspy-section-1">Section 1 Content</div> <div id="scrollspy-section-2">Section 2 Content</div> <div id="scrollspy-section-3">Section 3 Content</div>

API

Initiate

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js?load=spryJs"></script>
* var spryJsis automatically created and loads all components with default options. You can customize the variable in the src of the script.
<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script> <script> var spryObj = SpryJS('.scrollspy').scrollspy(); </script>
<script type="module"> import { scrollspy } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; const obj = scrollspy(); </script>

Update

This allows you to update the objects after Dom changes have been made without duplicating the event listeners. Useful with page builders and other dynamic Dom loading applications.

// Update Objects spryJs.scrollspyUpdate(); // OR Update All Components spryJs.update();
// Update Objects spryObj.scrollspyUpdate(); // OR Update All Components spryObj.update();
obj.update();
obj.update();

Destroy

This allows you to remove all event listeners and added functionality for the objects.

// Destroy Objects spryJs.scrollspyDestroy(); // OR Destroy All Components spryJs.destroy();
// Destroy Objects spryObj.scrollspyDestroy(); // OR Destroy All Components spryObj.destroy();
obj.destroy();
obj.destroy();

Options

scrollspy({ items: '.scrollspy', threshold: 200, progress: 'active', classActive: 'active', selectorAnchor: '[href*="#"],[data-scrollspy-section]', attributeAnchor: 'data-scrollspy-anchors', attributeClassActive: 'data-scrollspy-class-active', attributeSection: 'data-scrollspy-section', attributeThreshold: 'data-scrollspy-threshold', attributeProgress: 'data-scrollspy-progress' });
items?: string | Element[]

Selector or array of Elements for scrollspy containers. This will capture scroll events and activate anchors within the container when the section is visible within the viewport.

default: ".scrollspy"
threshold?: number

The threshold is the buffer in pixels before the anchor is considered active within the viewport.

default: 200

Override with attributeThreshold
progress?: string

Type of action to use when activating and deactivated anchors.

Options:
  • "active" (default)

    This will activate only one anchor at a time.

  • "linear"

    This will progress each anchor based on the position of the scroll. Essentially keeping all previous anchors active while you move forward and removing active anchors as you go back.

  • "seen"

    This will activate each anchor as it is seen, but does not remove any active anchors.

Override with attributeProgress
classActive?: string

The class that gets applied to the active anchor.

default: "active"

Override with attributeClassActive
selectorAnchor?: string

Selector for the Scroll Spy Anchors with Hashes within the Containers or attribute that matches attributeSection.

Any anchors found will then calculate the attributeSection first and if nothing is found then it will calculate the hash value from the href url. The url must be a valid url or relative path with a valid window.location.href.

default: '[href*="#"],[data-scrollspy-section]'

Override with attributeAnchor
attributeAnchor?: string

Allows you to use data attributes to specify your own selectors for the anchors within the containers. Uses a selector as its value.

default: "data-scrollspy-anchors"

Overrides selectorAnchor
attributeClassActive?: string

Data attribute to set the class that gets applied to the active anchor.

default: "data-scrollspy-class-active"

Overrides classActive
attributeSection?: string

This is an alternative to using selectorAnchor for your anchors. When using this attribute you can specify a selector for the anchor to connect with instead of using a hash and a section id="".

default: "data-scrollspy-section"
attributeThreshold?: string

Data attribute to set the Threshold for the anchors.

default: "data-scrollspy-threshold"

Overrides threshold
attributeProgress?: string

Data attribute to set the Progress type for the anchors.

default: "data-scrollspy-progress"

Overrides progress

Slider

These are basic lightweight slider. If you are looking for something more robust, you might want to check out Swiper or Glide

You can add loading="lazy" to the images, but it may be a little choppy when scrolling on some browsers. The Slider will detect the lazy loading images and pre load them on the next and previous slides when using SprySliderJS. If you want a smooth transition it is best to load all the images initially.

<!-- Slider No JS --> <div class="slider" data-snap> <div class="slider-slides"> <img id="no-js-slide1" src="/placeholder-1.jpg" loading="lazy"> <img id="no-js-slide2" src="/placeholder-2.jpg" loading="lazy"> <img id="no-js-slide3" src="/placeholder-3.jpg" loading="lazy"> </div> </div> <!-- Slider With SprySliderJS --> <div class="slider" data-snap> <button class="slider-prev link icon prev circle"></button> <button class="slider-next link icon next circle"></button> <div class="slider-slides"> <img src="/placeholder-1.jpg" loading="lazy"> <img src="/placeholder-2.jpg" loading="lazy"> <img src="/placeholder-3.jpg" loading="lazy"> </div> <div class="slider-pagination p-3"></div> </div>

Classes
slider Main Slider Container
slider-slides Slides Wrapper
slider-prev Previous Navigation Button. Used with SprySliderJS
slider-next Next Navigation Button. Used with SprySliderJS
slider-pagination Pagination Navigation. Used with SprySliderJS
{color} Adds Accent color to component

Data Options
data-snap Forces stopping on each slide even when using grid layout. This is not forced with momentum scrolling.
data-loop Allows the Slides to Loop. Requires SprySliderJS
data-play="{n}" Enable autoplay in milliseconds. Requires SprySliderJS
data-stop="{enum}" Pauses Autoplay. Requires SprySliderJS

Options:

  • hover - Pause autoplay while hovering over slider
  • action - Pause autoplay while hovering over a link, button or making a text selection within slider.

Toggle

Toggle are a way to toggle an element ON or OFF
Loading Toggle
<script type="module"> import { toggle } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; toggle({options}); </script>

* toggle() can be called multiple times without creating duplicated event listeners and other side effects. toggle() will ignore any pre-registered toggles. This makes it compatible when using it with page builders or other dynamically created html blocks.

Toggle Example.
<!-- Toggle Activator --> <button data-toggle="next"></button> <!-- Toggle Element --> <div>...</div>

Options
classOpen The class that gets applied to an opened toggled element
default: open
classActive The class that gets applied to an opened toggled activator
default: active

Toggle Attributes
data-toggle="{selector}" selector can be:
  • Any css selector.

    See Special Selector Values for additional values.

  • next which will be the nextElementSibling
  • prev which will be the previousElementSibling
  • {empty} If empty or excluded then it will toggle itself.
data-toggle-close="{selector}" This runs before data-toggle="{selector}" and force closes elements.

selector can be:
  • Any css selector

    See Special Selector Values for additional values.

  • next which will be the nextElementSibling
  • prev which will be the previousElementSibling
  • {empty} If empty or excluded then it will close the closest opened toggled element, including itself.
data-toggle-open="{selector}" This runs before data-toggle="{selector}" and force opens elements.

selector can be:
  • Any css selector

    See Special Selector Values for additional values.

  • next which will be the nextElementSibling
  • prev which will be the previousElementSibling
  • {empty} If empty or excluded then it will open the closest closed toggled element, including itself.
data-toggle-dismissible This will allow you to click outside the Toggle Element or press the escape key to close the toggle.
data-toggle-escapable This will allow you to press the escape key to close the toggle.
data-toggle-close Closes the closest Opened Toggled Element.
data-toggle-timeout="{n}" This will auto close the toggle element in the milliseconds.
{n} is the time in milliseconds

Special Selector Values
{closest-selector} Starting your selector with {...} will use its value to select the closest node.

Example:
{.container} ul li button
Is equivalent to: this.closest('.container').querySelectorAll('ul li button')
{n} Using {n} will get replaced with the current nodes index value.

Example:
{.container} ul li:nth-child({n}) > div
Is equivalent to:
var nodeIndex = Array.from(this.parentElement.children).indexOf(this);
this.closest('.container').querySelectorAll('ul li:nth-child('+(nodeIndex+1)+') > div')

Hash

SpryJS Hash is not for encryption or secured data. If you need to ensure data safety or a stronger digest then you may want to look at SubtleCrypto
SpryJS Hash is for basic hashing and requires unicode safe characters. Objects will be converted to JSON strings before they are hashed.

Using Hash

<script type="module"> import { hash } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; /** * Get an MD5 hash * * @param inputString: string | number | object - The value to hash. Objects will be converted to JSON strings before they are hashed. * * @returns string */ const myHash = hash.md5(inputString: string | number | object); /** * Get an SHA-256 hash * * @param inputString: string | number | object - The value to hash. Objects will be converted to JSON strings before they are hashed. * * @returns string */ const myHash = hash.sha256(inputString: string | number | object); </script>

Query

The query object is a simple lightweight tool to help you with some basic dom manipulations.
Query is only intended for collections and not single items.

Using Query

<script type="module"> import { query } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; // Add Class and Attribute to Elements query('.selector').addClass('stuff').attr('data-stuff', 'things'); // Loop Elements query('.selector').each((el, index) => { console.log(index, el); }); // Add Click Events query('.selector').on('click', (el, index, event) => { console.log(index, el, event); }); // Get Elements var q = query('.selector'); console.log(q.elements); </script>

Return Values

// Collection { elements, ...methods }

* Methods can be chained as the response always returns the updated collection.

Query Methods
nth(
n?: number
) 		Returns a new collection with only the matching element.
{n} Can be negative for reverse lookup
first()
 Returns a new collection with only the first element.
Is alias of nth(1)
last()
 Returns a new collection with only the last element.
Is alias of nth(-1)
slice(
start: number,
end?: number
) 		Returns a new collection from the start and end.
start and end can be negative for reverse lookup
each(
callback: Callable
) 		Loops through each element found in the collection and runs the callback.
The callback will provide the (element, index) of each item in the collection.
toggleClass(
className: string,
force?: boolean
) 		Toggles the class of each element found in the collection.
addClass(
className: string
) 		Adds the class to each element found in the collection.
removeClass(
className: string
) 		Removes the class from each element found in the collection.
toggleAttr(
attributeName: string,
force?: boolean
) 		Toggles the attribute of each element found in the collection.
attr(
attributeName: string,
attributeValue: string
) 		Sets the attribute of each element found in the collection.
removeAttr(
attributeName: string
) 		Removes the attribute from each element found in the collection.
on(
action: string,
callback: Callable (
element?: Element,
index?: number,
event?: Event
)
) 		Adds an event listener to each element found in the collection.
The action must be a compatible event type with addEventlistener.
off(
action: string
listener: Callable
) 		Removes an event listener on each element found in the collection.
The action must be a compatible event type with addEventlistener.
The listener must be a compatible with EventListenerOrEventListenerObject.
