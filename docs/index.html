SpryJS

SpryJS

SpryJS is a simple and lightweight JS Toolkit that pairs nicely with SpryCSS, but can be used all on its own.
The goal is to provide general functionality and utilities to speed up your development.

Support

You can report issues or make feature requests on the GitHub Issues page.

If you like to help us grow, then please add a star on the SpryJS GitHub Repository or by sharing us on social media.

Installation

CDN Default ESM:

<script type="module"> import Spry from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; // Cookie Spry.cookie.set(...); Spry.cookie.get(...); Spry.cookie.remove(...); // Hash Spry.hash.md5(...); Spry.hash.sha256(...); // Query Spry.query(...).addClass(...); // Load Single Components Spry.navigable(...); Spry.observer(...); Spry.parallax(...); Spry.scrollspy(...); Spry.slider(...); Spry.toggle(...); // Update Single Components Spry.navigableUpdate(); Spry.observerUpdate(); Spry.parallaxUpdate(); Spry.scrollspyUpdate(); Spry.sliderUpdate(); Spry.toggleUpdate(); // Destroy Single Components Spry.navigableDestroy(); Spry.observerDestroy(); Spry.parallaxDestroy(); Spry.scrollspyDestroy(); Spry.sliderDestroy(); Spry.toggleDestroy(); // Load all Components Spry.load(); // Update all Components Spry.update(); // Destroy all Components Spry.destroy(); </script>

CDN Modules ESM:

<script type="module"> import { cookie, hash, query, navigable, observe, parallax, scrollspy, slider, toggle, } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; // Cookie cookie.set(...); cookie.get(...); cookie.remove(...); // Hash hash.md5(...); hash.sha256(...); // Query query(...).addClass(...); // Load Components navigable(...); observer(...); parallax(...); scrollspy(...); slider(...); toggle(...); // Update Components const nav = navigable(...); nav.update(); // Destroy Components const nav = navigable(...); nav.destroy(); </script>

Cookie

Using Cookie

<script type="module"> import { cookie } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; /** * Set a Cookie * * @param cookieName string - The cookie name to set * @param cookieValue any - The value * @param expireSeconds number - Number in seconds to expire. 0 = Session expire. * * @returns void */ cookie.set(cookieName: string, cookieValue: any, expireSeconds: number); /** * Get a Cookie * * @param cookieName string - The name of the cookie to get * * @returns string */ cookie.get(cookieName: string); /** * Remove a Cookie * * @param cookieName string - The cookie name to remove. This will reset the cookie to expire immediately. * * @returns void */ cookie.remove(cookieName: string); </script>

Hash

SpryJS Hash is not for encryption or secured data. If you need to ensure data safety or a stronger digest then you may want to look at SubtleCrypto
SpryJS Hash is for basic hashing and requires unicode safe characters. Objects will be converted to JSON strings before they are hashed.

Using Hash

<script type="module"> import { hash } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; /** * Get an MD5 hash * * @param inputString: string | number | object - The value to hash. Objects will be converted to JSON strings before they are hashed. * * @returns string */ const myHash = hash.md5(inputString: string | number | object); /** * Get an SHA-256 hash * * @param inputString: string | number | object - The value to hash. Objects will be converted to JSON strings before they are hashed. * * @returns string */ const myHash = hash.sha256(inputString: string | number | object); </script>

Navigable

This allows <a> and <button> to be Navigable via the Keyboard Arrow keys and clickable via the Keyboard Space and Enter keys. It also adds ability to use the Escape key to ignore focus.
<script src="https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.js"></script>
<script type="module"> import { navigable } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; </script>
<script type="module"> import { navigable } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; navigable(); // or navigable('.navigable', { anchorSelector: 'a, button, input, [tabindex]' }); </script>

* navigable() can be called multiple times without creating duplicated event listeners and other side effects. It will ignore any pre-registered elements. This makes it compatible when using it with page builders or other dynamically created html blocks.

Example:
<nav class="navigable"> <ul> <li><a href="#">Link 1</a></li> <li><a href="#">Link 2</a></li> <li><a href="#">Link 3</a></li> </ul> </nav>

Arguments
selector?: string Class Name for navigable items.
default: ".navigable"

This will capture keyboard events within this element and navigate between the anchors
options?: object
  • anchorSelector?: string

    Selector for tracking navigable items within the container.

    default: "a, button, input, [tabindex]"

Data Attributes
data-navigable-anchors

Selector for tracking navigable items within the navigable container.

This will override the default options.

Observer

Loading Observer
<script type="module"> import { observer } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; observer({options}); </script>

* observer() can be called multiple times without creating duplicated event listeners and other side effects. it will ignore any pre-registered elements. This makes it compatible when using it with page builders or other dynamically created html blocks.

Example:
<!-- Observe Element --> <div class="observe"> ... </div>

Options
classObserve Class Name for elements to be observed.
default: observe

The observer will watch for these elements to be visible in the viewport.
classObserving Class that gets toggled when elements are visible within the viewport.
default: observing

The observer will add this class for these elements that are currently visible in the viewport.
classObserved Class that gets added when elements become visible within the viewport.
default: observed

This class gets added when the element becomes visible within the viewport. This class does not get removed.
rootMargin Bounding box before computing intersections. Negative values will shrink the bounding box. Each side value is required.
default: 0px 0px 0px 0px
Learn more about rootMargin
threshold Either a single number from 0 - 1 or an array of numbers which indicate at what percentage of the target's visibility the observer's callback should be executed.
default: 0
Learn more about threshold
delay Delay in milliseconds before the Observer acknowledges the observed element.
default: 100

If multiple elements are observed at the same time then the delay will be stacked by the element index

Parallax

Loading Parallax
<script type="module"> import { parallax } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; parallax({options}); </script>

* parallax() can be called multiple times without creating duplicated event listeners and other side effects. parallax() will ignore any pre-registered elements. This makes it compatible when using it with page builders or other dynamically created html blocks.

Example:
<!-- Parallax Container --> <div style="max-height: 200px; overflow: hidden;"> <!-- Parallax Element --> <div><!-- Large Content Here --></div> </div>

Options
classParallax Class Name for main parallax items.
default: parallax

This will parallax the elements translate property. However, hiding the overflow and all other styles will need to be applied separately. A parallax element must be contained in a parent element that is smaller than the parallax element.
classParallaxBackground Class Name for background position parallax items.
default: parallax-background

This will parallax the elements background-position property. However, all other styles will need to be applied separately. Background image must be larger than the element to work properly.
classParallaxHorizontal Class Name for changing the parallax direction to Horizontal.
default: parallax-horizontal
minWidth Minimum width in pixels of the window to apply effects. Useful if you do not what to apply parallax for smaller devices.
default: 0
threshold The threshold is the buffer in pixels before the parallax element starts to animate before it reaches the viewport. This helps reduce flickering when the element animates from baseline position. You can use a positive number to prevent animation while partially in viewport.
default: -300
delay Number in milliseconds to apply easing delay. This allows for a smoother animation on non-smooth scrolling devices and might need adjustment based on your needs. Set to 0 to disable.
default: 300

Query

The query object is a simple lightweight tool to help you with some basic dom manipulations.
Query is only intended for collections and not single items.

Using Query

<script type="module"> import { query } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; // Add Class and Attribute to Elements query('.selector').addClass('stuff').attr('data-stuff', 'things'); // Loop Elements query('.selector').each((el, index) => { console.log(index, el); }); // Add Click Events query('.selector').on('click', (el, index, event) => { console.log(index, el, event); }); // Get Elements var q = query('.selector'); console.log(q.elements); </script>

Return Values

// Collection { elements, ...methods }

* Methods can be chained as the response always returns the updated collection.

Query Methods
nth(
n?: number
) 		Returns a new collection with only the matching element.
{n} Can be negative for reverse lookup
first()
 Returns a new collection with only the first element.
Is alias of nth(1)
last()
 Returns a new collection with only the last element.
Is alias of nth(-1)
slice(
start: number,
end?: number
) 		Returns a new collection from the start and end.
start and end can be negative for reverse lookup
each(
callback: Callable
) 		Loops through each element found in the collection and runs the callback.
The callback will provide the (element, index) of each item in the collection.
toggleClass(
className: string,
force?: boolean
) 		Toggles the class of each element found in the collection.
addClass(
className: string
) 		Adds the class to each element found in the collection.
removeClass(
className: string
) 		Removes the class from each element found in the collection.
toggleAttr(
attributeName: string,
force?: boolean
) 		Toggles the attribute of each element found in the collection.
attr(
attributeName: string,
attributeValue: string
) 		Sets the attribute of each element found in the collection.
removeAttr(
attributeName: string
) 		Removes the attribute from each element found in the collection.
on(
action: string,
callback: Callable (
element?: Element,
index?: number,
event?: Event
)
) 		Adds an event listener to each element found in the collection.
The action must be a compatible event type with addEventlistener.
off(
action: string
listener: Callable
) 		Removes an event listener on each element found in the collection.
The action must be a compatible event type with addEventlistener.
The listener must be a compatible with EventListenerOrEventListenerObject.

Scroll Spy

Loading Scroll Spy
<script type="module"> import { scrollspy } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; scrollspy({options}); </script>

* scrollspy() can be called multiple times without creating duplicated event listeners and other side effects. This will ignore any pre-registered elements. This makes it compatible when using it with page builders or other dynamically created html blocks.

Example:
<!-- ScrollSpy Container --> <div class="scrollspy"> <!-- ScrollSpy Anchors --> <a href="#scrollspy-section-1">Section 1 Anchor</a> <a href="#scrollspy-section-2">Section 2 Anchor</a> <a href="#scrollspy-section-3">Section 3 Anchor</a> </div> <!-- ScrollSpy Sections --> <div id="scrollspy-section-1">Section 1 Content</div> <div id="scrollspy-section-2">Section 2 Content</div> <div id="scrollspy-section-3">Section 3 Content</div>

Options
containerSelector Selector for the Scroll Spy Containers.
default: .scrollspy
anchorHashSelector Selector for the Scroll Spy Anchors with Hashes within the Containers.
default: [href*="#"]

Any anchors found will then calculate the hash value from the href url. The url must be a valid url or relative path with a valid window.location.href. If this is not possible then you might need to use the anchorDataAttribute instead.
anchorDataAttribute This is an alternative method instead of using the anchorHashSelector. This will allow you to use data attributes to specify your own selectors for the anchors within the containers. Uses a selector as its value.
default: data-scrollspy
classActive The class that gets applied to the active anchor.
default: active
threshold The threshold is the buffer in pixels before the anchor is considered active within the viewport.
default: 200
progress Type of action to use when activating and deactivated anchors.
Options:
  • active (default)

    This will activate only one anchor at a time.

  • linear

    This will progress each anchor based on the position of the scroll. Essentially keeping all previous anchors active while you move forward and removing active anchors as you go back.

  • seen

    This will activate each anchor as it is seen, but does not remove any active anchors.

Sliders

These are basic lightweight sliders. If you are looking for something more robust, you might want to check out Swiper or Glide

You can add loading="lazy" to the images, but it may be a little choppy when scrolling on some browsers. The Slider will detect the lazy loading images and pre load them on the next and previous slides when using SprySliderJS. If you want a smooth transition it is best to load all the images initially.

<!-- Slider No JS --> <div class="slider" data-snap> <div class="slider-slides"> <img id="no-js-slide1" src="/placeholder-1.jpg" loading="lazy"> <img id="no-js-slide2" src="/placeholder-2.jpg" loading="lazy"> <img id="no-js-slide3" src="/placeholder-3.jpg" loading="lazy"> </div> </div> <!-- Slider With SprySliderJS --> <div class="slider" data-snap> <button class="slider-prev link icon prev circle"></button> <button class="slider-next link icon next circle"></button> <div class="slider-slides"> <img src="/placeholder-1.jpg" loading="lazy"> <img src="/placeholder-2.jpg" loading="lazy"> <img src="/placeholder-3.jpg" loading="lazy"> </div> <div class="slider-pagination p-3"></div> </div>

Classes
slider Main Slider Container
slider-slides Slides Wrapper
slider-prev Previous Navigation Button. Used with SprySliderJS
slider-next Next Navigation Button. Used with SprySliderJS
slider-pagination Pagination Navigation. Used with SprySliderJS
{color} Adds Accent color to component

Data Options
data-snap Forces stopping on each slide even when using grid layout. This is not forced with momentum scrolling.
data-loop Allows the Slides to Loop. Requires SprySliderJS
data-play="{n}" Enable autoplay in milliseconds. Requires SprySliderJS
data-stop="{enum}" Pauses Autoplay. Requires SprySliderJS

Options:

  • hover - Pause autoplay while hovering over slider
  • action - Pause autoplay while hovering over a link, button or making a text selection within slider.

Toggles

Toggles are a way to toggle an element ON or OFF
Loading Toggles
<script type="module"> import { toggle } from 'https://cdn.jsdelivr.net/gh/ggedde/spry-js@1.0.0/dist/spry.mjs'; toggle({options}); </script>

* toggle() can be called multiple times without creating duplicated event listeners and other side effects. toggle() will ignore any pre-registered toggles. This makes it compatible when using it with page builders or other dynamically created html blocks.

Toggle Example.
<!-- Toggle Activator --> <button data-toggle="next"></button> <!-- Toggle Element --> <div>...</div>

Options
classOpen The class that gets applied to an opened toggled element
default: open
classActive The class that gets applied to an opened toggled activator
default: active

Toggle Attributes
data-toggle="{selector}" selector can be:
  • Any css selector.

    See Special Selector Values for additional values.

Special Selector Values
{closest-selector} Starting your selector with {...} will use its value to select the closest node.

Example:
{.container} ul li button
Is equivalent to: this.closest('.container').querySelectorAll('ul li button')
{n} Using {n} will get replaced with the current nodes index value.

Example:
{.container} ul li:nth-child({n}) > div
Is equivalent to:
var nodeIndex = Array.from(this.parentElement.children).indexOf(this);
this.closest('.container').querySelectorAll('ul li:nth-child('+(nodeIndex+1)+') > div')
